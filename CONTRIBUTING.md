## üìù Reporting Issues

Before opening a new issue:

1. Check if the issue already exists  
2. Provide clear steps to reproduce the problem (if applicable)  
3. Include screenshots or error messages  
4. Mention your environment (OS, Node.js version, etc.)

---

## üîß Submitting Pull Requests (PRs)

To submit a PR safely:

1. **Fork** the repository  
2. **Clone** your fork  
3. Create a new branch:
   ```sh
   git checkout -b feature/your-feature-name
   ```
4. Make your changes (ensure they are **educational/demo only**)  
5. Commit your changes with a clear message:
   ```sh
   git commit -m "Docs: Improved README instructions for Node.js setup"
   ```
6. Push your branch:
   ```sh
   git push origin feature/your-feature-name
   ```
7. Open a Pull Request with a clear description of your contribution
